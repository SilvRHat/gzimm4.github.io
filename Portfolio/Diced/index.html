Game Description


A stylized pizza-making game where 4 players compete for the top score on constructing outrageous pizzas. Players battle in kitchens equipped with stand mixers, cutting boards, stovetops, and a grand pizza oven. Every round a kitchen, theme, and recipe are chosen. Submissions are scored on dough ratios, correct toppings, cooking job, and hidden bonuses. Claiming wins, players can refine their knife skills, speed, and chef-know-how.

Supported on: PC, Tablet, and Mobile
Powered by Roblox

Structure


Provided are descriptions on scripts which directly guide gameplay.
Client-sided scripts typically monitor user input, provide immediate feedback, and signal server to validate and carry out those actions.
Server-sided scripts are take on various roles and communicate data through signaled functions.

Main [SERVER]

Operates gameplay cycle and game state. Cycles through setup, intermission, game round, round results and rewards.
Also manages endpoints to other server scripts to handle purchasable content which affects round setup (Ex. Choose Recipe Product).


Store [SERVER]

Handles player data and product purchases. Enables player data to persist across game sessions. Enables a daily bonus rewarding extra points for consecutive days a player visits game. Manages upgrade purchases and purchasable content, verifying purchase are valid and fulfilled.



Recipe Module [MOD - SERVER]

Operates gameplay systems adjusted towards a specific recipe. Verifies and performs interactions, manipulating 3D environment in process. Communicates with Main, tracking most current scores and submission entries.



Recipe Module [MOD - CLIENT]

Operates gameplay systems adjusted towards a specific recipe. Prompts player when interactions can be made. Sends signal to server on input 'E'. Some interactions require a key be pressed, where progress is displayed to user and communicated with server on early cancels.



Appearance [SERVER]

Manages minor details in game appearance. Customizes characters in a chef avatar, designating premium players with red chat text and red clothing.



UI_main [CLIENT]

Implements entire user interface. Communicates game state with player, round details, and round results. Allows user to interact with a shop, a tutorial, and current selected recipe.
