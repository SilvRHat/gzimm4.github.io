Game Description


A stylized pizza-making game where 4 players compete for the top score on constructing outrageous pizzas. Players battle in kitchens equipped with stand mixers, cutting boards, stovetops, and a grand pizza oven. Every round a kitchen, theme, and recipe are chosen. Submissions are scored on dough ratios, correct toppings, cooking job, and hidden bonuses. Claiming wins, players can refine their knife skills, speed, and chef-know-how.

Supported on: PC, Tablet, and Mobile
Powered by Roblox

Structure


Provided are significant scripts which directly guide gameplay. Client scripts typically monitor user input, provide immediate feedback, and signal server to validate and carry out completely. Server scripts are split into various roles and communicate data through signaled functions.

Main [SERVER SIDED]
Operates gameplay cycle and game state. Cycles through setup, intermission, game round, round results and rewards.
  Setup - Selects map and recipe module to play. Loads these into world.
  Intermission - Brief sleep to introduce round conditions to players
  Game round - Activates gameplay scripts and movement. Waits for all submissions or round time to reach 0. Deactivates gameplay scripts and movement.
  Rewards - Scores each submission. Teleports losing player to lobby server. Rewards points to submissions and a bonus to winner.

Linked with: UI_main, Store


Store [SERVER SIDED]
Handles player data and product purchases. Enables player data to persist across game sessions. Enables a daily bonus rewarding extra points for consecutive days a player visits game. Manages upgrade purchases and purchasable content, verifying purchase are valid and fulfilled.

Linked with: Main, UI_main


Recipe Module [MODULE | PARALLEL] (x12)
Operates gameplay systems adjusted towards a specific recipe.   Client Side - Prompts player when interactions can be made. Sends signal to server on input 'E'. Some interactions require a key be pressed, where progress is displayed to user and communicated with server on early cancels.
  Server Side - Verifies and performs interactions, manipulating 3D environment in process. Communicates with Main, specifying most current scores and submission entries.

Required from: Main, UI_main Appearance [SERVER SIDED]
Manages minor details in game appearance. Customizes characters in a chef avatar, designating premium players with red chat text and red clothing.


UI_main [CLIENT SIDED]
Implements entire User interface. Signaled from main, UI main communicates game state with player, round details, and round results. Allows user to interact with a shop, a tutorial, and current selected recipe.

Communicates with: Main, Store
