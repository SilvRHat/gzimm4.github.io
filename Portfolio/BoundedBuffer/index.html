Exposition


The bounded buffer problem faces managing shared memory between multiple threads in a safe manner. This project is a solution to project assignment 3 for Operating Systems at University of Colorado, Boulder.

Goal


Design a multi-threaded application, written in C, which resolves host names to corresponding IP addresses. The application should implement two classes of threads, requesters and resolvers. Requester threads will read hostnames from a number of input files and place each hostname in a shared array of limited size (the bounded buffer). Resolver threads will take hostnames from the shared array and perform a DNS lookup and write to an output file.

Man Page

NAME

multi-lookup - resolve a set of hostnames to IP addresses

SYNOPSIS

multi-lookup <# requester> <# resolver> <requester log> <resolver log> [ <data file> ... ]

DESCRIPTION

The file names specified by <data file> are passed to the pool of requester threads which place information into a shared data area. Resolver threads read the shared data area and find the corresponding IP address.

<# requesters> number of requestor threads to place into the thread pool.

<# resolvers> number of resolver threads to place into the thread pool.

<requester log> name of the file into which all the requester status information is written.

<resolver log> name of the file into which all the resolver status information is written.

<data file> filename to be processed. Each file contains a list of host names, one per line, that are to be resolved.

Constraints


  • Bounded buffer size - 20 Elements
  • Cannot use pre-existing thread-safe queue or file io libraries
  • Built with “-Wall” and “-Wextra” producing no errors or warnings
  • Run with leaking no memory
  • Cannot use global variables for worker threads
  • Synchronization and Deadlock - Program should synchronize 3 shared resources avoiding deadlock and busy waiting.
